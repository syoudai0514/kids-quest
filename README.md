# ほしぞらクエスト 🚀⭐🦕

5歳（年長）向けの「毎日ミッション学習アプリ」。宇宙と恐竜を旅しながら、
相棒モンスター **ホッシュ** といっしょに「よむ・かく・すうじ」を学びます。
「勉強」ではなく「あそびの延長」に感じられることを最優先に設計しています。

> Android タブレットのブラウザで動く **React + PWA（オフライン対応）**。
> アカウント登録不要・データは端末内のみに保存。

---

## いま動くもの（MVP）

依頼の「まずほしいもの」を満たした、動く骨組みです。

- ✅ **3つの分野（小1レベルまで）** … すべてプレイ可能：
  - **よむ** … 絵⇄ことばのマッチング（約80語）＋**小1漢字の読み**（48字）。
  - **かく** … 指でなぞる文字書き。ひらがな・カタカナ全部＋**小1漢字**。お手本→自由書き。
  - **すうじ** … かぞえる／大小くらべ（100まで）／**くり上がり・くり下がり**の足し引き（〜20）／10をつくる／数のならび。
- ✅ **モンスター約100体** … 相棒ホッシュ＋集める仲間。SVGの独自デザイン（絵柄8種×多色）。息抜きバトルで捕まえて図鑑を埋める。
- ✅ **毎日ミッションの骨組み** … コア5タスク（各4問・1〜2分）。終わると「おかわり」枠。
- ✅ **難易度の自動調整** … 連続正解でレベルアップ／間違えても責めず、ヒントを増やし必ずできるよう支える。
- ✅ **ご褒美システム** … 累計クリアで惑星が解放され、新しいなかま（モンスター）に出会い、物語が進む。
- ✅ **息抜きバトル（簡易版）** … オリジナルモンスターのターン制バトル。**1日の基本プレイ上限**つき。
      上限を超えたいときは「ついか問題」を解いて **解放チケット🎟️** をためる動機設計。
- ✅ **音声読み上げ（日本語TTS）** … 指示・問題文・正誤を必ず声でも伝える。
- ✅ **保護者ビュー** … 今日やった量・得意/苦手の傾向・息抜き解放の回数・設定。

すべて画像/音声ファイル不要（モンスターはSVG、効果音はWeb Audioで合成）なので、
**完全オフライン**で動きます。

## うごかし方

```bash
npm install
npm run dev        # 開発サーバ
# または
npm run build && npm run preview   # 本番ビルドの確認
```

Android タブレットで使うときは、ビルドを HTTPS で配信し、ブラウザの
「ホーム画面に追加」でインストールすると全画面のアプリとして動きます。

PWAアイコンを作り直す場合:

```bash
node scripts/make-icons.mjs   # public/icon-192.png, icon-512.png を生成
```

## 設計（拡張しやすさ重視）

データ駆動で、**新しい分野・問題・モンスター・惑星は配列に足すだけ**で増やせます。

```
src/
├─ engine/
│  ├─ tts.js          日本語の音声読み上げ
│  ├─ sfx.js          効果音（Web Audioで合成。ファイル不要）
│  ├─ storage.js      localStorage 永続化＋日付ロールオーバー
│  ├─ difficulty.js   難易度の自動調整（分野ごとの習熟度モデル）
│  ├─ activities.js   学習分野(ドメイン)のレジストリ ← ここに分野を足す
│  └─ missions.js     毎日ミッションの組み立て
├─ data/
│  ├─ monsters.js     オリジナルモンスター定義 ← ここに足す
│  ├─ planets.js      惑星(ワールド)進行 ← ここに足す
│  └─ content/
│     └─ reading.js   「よむ」の単語プール＋出題ジェネレータ
├─ components/
│  ├─ Monster.jsx     モンスターのSVG描画
│  └─ common.jsx      星背景・紙吹雪・読み上げボタン等
├─ screens/
│  ├─ HomeScreen.jsx        ハブ
│  ├─ ActivityPlayer.jsx    全分野共通の出題プレイヤー
│  ├─ BattleScreen.jsx      息抜きバトル
│  ├─ CollectionScreen.jsx  なかまずかん
│  ├─ ParentScreen.jsx      保護者ビュー
│  └─ CelebrationOverlay.jsx ごほうび演出
└─ state/
   └─ GameContext.jsx  ゲーム全体の状態（Context + Reducer）
```

### 拡張のしかた

**新しい「よむ」の問題を足す** → `src/data/content/reading.js` の `WORDS` に1行追加:

```js
{ text: 'ぎんが', emoji: '🌌', kana: 'hira', tier: 4, theme: 'space' }
```

**新しいモンスターを足す** → `src/data/monsters.js` の `MONSTERS` に1件追加
（`art` は `blob|dino|star|rock|ghost`、`role` は `partner|friend|wild`）。

**新しい惑星を足す** → `src/data/planets.js` の `PLANETS` に1件追加
（`unlockAt` に解放する累計クリア数、`unlockMonster` に出会うモンスターID）。

**新しい分野を足す**（「かく」「すうじ」と同じやり方）:
1. `src/data/content/` に generator（`generateQuestion(params)`）を作る。
   - 選択式なら `type:'choice'`（戻り値は `reading.js` / `numbers.js` と同じ形）。
   - なぞり書きのような独自UIなら `type:'trace'` 等を返し、`ActivityPlayer` に分岐を足す
     （「かく」は `components/TracingCanvas.jsx` を使用。文字はフォントから形を生成するので
     文字データ不要、`writing.js` のプールに足すだけ）。
2. `src/engine/activities.js` のドメインに `available:true` と `generateQuestion` を設定。
3. ミッション・難易度調整・保護者ビューには**自動で乗ります**。

> コンテンツを大きく更新したら `src/state/GameContext.jsx` の `CONTENT_VERSION` を上げると、
> 進捗（習熟度・収集・累計）を保ったまま「今日のミッション」を新内容で作り直します。

### 難易度ロジックの要点（`difficulty.js`）

- **得意分野**: 3連続正解で `level` が上がり、選択肢が増え・カタカナや長い語が出る。
- **苦手分野**: 連続ミスで `level` をそっと半段だけ下げ、`hintLevel` を上げて支える。
- **その場で必ずできる**: 同じ問題で2回つまずくと、正解の選択肢を光らせて成功体験につなげる
  （難易度には「最初の1回の結果」だけを反映＝やり直しは減点しない）。

## 著作権について

モンスター・世界観はすべて**独自デザイン**です。既存IP（ポケモン等）の画像・名称・
データは一切使用していません。バトルは「捕まえる・育てる・対戦する」楽しさを
オリジナルで再現しています。
